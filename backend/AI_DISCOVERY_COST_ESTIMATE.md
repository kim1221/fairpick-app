## 🆚 **기존 방식 (Naver API) vs AI 직접 방식**

### 기존 방식 (admin-hot-discovery.ts)
- **네이버 API 호출**: 1,000회/일 × 30일 = 30,000회/월
- **Gemini AI 호출**: 배치 처리 (3회 × 30일) = 90회/월
- **월 비용**: **~$9.90** (₩14,000)
- **문제점**: 
  - 낮은 정확도 (옷가게, 교회, 지난 이벤트 등)
  - 복잡한 파이프라인 (검색 → 필터링 → 추출 → 검증 → 정규화)
  - Rate Limit 429 에러 빈번

### AI 직접 방식 (NEW)
- **Gemini AI 호출**: 15회/일 (팝업) + 1회/주 (평가) = **~450회/월**
- **월 비용**: **~$0.46** (₩650)
- **장점**:
  - ✅ **95% 비용 절감** ($9.90 → $0.46)
  - ✅ **훨씬 높은 정확도** (AI가 직접 웹 검색 + 판단)
  - ✅ **DB 중복 체크** (자동으로 새로운 것만 추천)
  - ✅ **간단한 파이프라인** (AI 호출 → DB 저장)
  - ✅ **Rate Limit 없음** (Gemini API 한도: 1,500 RPM)

## ✅ **결론**

### 💡 **AI 직접 방식 선택 이유**
1. **비용**: 기존 대비 **95% 절감** ($9.90 → $0.46)
2. **정확도**: 환각 없음, 실제 웹 검색 기반
3. **유지보수**: 복잡한 파이프라인 제거
4. **확장성**: 질문 늘려도 비용 부담 없음
5. **법적 안전성**: Gemini Grounding은 공식 웹 검색 (크롤링 아님)
