# PROJECT_CONTEXT

## Project Overview

Fairpick은 한국의 문화 이벤트(전시, 공연, 축제, 팝업)를 추천하는 서비스입니다.

이 프로젝트의 가장 큰 특징은
**사용자 행동 데이터가 거의 없는 초기 단계(Cold Start)에서도**
“지금 사회적으로 주목받는 이벤트”를 안정적으로 추천하는 데 있습니다.

Fairpick은
“내 취향에 맞는 것”보다 먼저
**“지금 사람들이 실제로 이야기하고 있는 것”을 놓치지 않게 하는 서비스**입니다.

---

## Service Identity

Fairpick은 이벤트의 ‘핫함’을
개인의 취향이나 내부 추정이 아니라,
**외부에서 관측 가능한 객관적 신호**로 판단합니다.

우리는 다음 질문에 답하려고 합니다:

> “이 이벤트는 지금, 실제 세상에서 주목받고 있는가?”

이를 위해 단일 출처나 단일 알고리즘에 의존하지 않고,
여러 신호를 조합해 **현실과 최대한 정합적인 판단**을 목표로 합니다.

---

## Core Principles

### 1. Cold Start First
- 사용자 데이터가 없어도 추천이 가능해야 한다
- 추천 품질이 “데이터 축적 이후”로 미뤄지지 않아야 한다

### 2. Objective Signals over Subjective Curation
- 개인의 감이나 취향 기반 큐레이션을 최소화한다
- 측정 가능하고 설명 가능한 신호를 우선한다

### 3. Multi-Source Truth
- 하나의 데이터 소스만으로 “핫함”을 판단하지 않는다
- 공공 데이터, 검색 트렌드, 구조적 속성, AI 분석을 조합한다

### 4. Category-Aware Hotness
- 모든 이벤트를 같은 기준으로 평가하지 않는다
- 공연, 전시, 축제, 팝업은 “핫함”의 의미가 서로 다르다

### 5. Admin-Assisted Discovery
- 완전 자동화는 목표가 아니다
- AI는 “발견과 제안”, 사람은 “판단과 확정”을 담당한다

---

## Conceptual Data Model (개념 수준)

Fairpick은 다음과 같은 개념적 엔티티를 중심으로 동작합니다.

- **Event**
  - 여러 출처에서 수집된 정보를 하나로 정규화한 이벤트 개체
  - 서비스에 실제로 노출되는 유일한 이벤트 단위

- **Hot Candidate**
  - AI나 자동화 로직이 발견했지만,
    아직 사람의 검증을 거치지 않은 이벤트 후보

- **User Signal**
  - 사용자가 명시적으로 표현한 관심 (저장, 클릭 등)
  - 초기에는 중요도가 낮고, 장기적으로 확장 대상

이 프로젝트는
“이벤트 하나 = 점수 하나”가 아니라,
**여러 신호의 조합으로 이벤트를 이해하는 구조**를 지향합니다.

---

## Recommendation Philosophy

Fairpick의 추천은 두 단계로 구분됩니다.

### 1. Featured Recommendation (지금 추천)
- 모든 사용자에게 공통으로 제공되는 추천
- “지금 이 시점에 객관적으로 핫한 이벤트”가 기준
- 초기 서비스의 핵심 가치

### 2. Personalized Recommendation (향후 확장)
- 사용자 행동 데이터를 기반으로 한 개인화 추천
- 데이터가 충분히 쌓인 이후 단계적으로 도입

초기 Fairpick의 성공 여부는
**Featured Recommendation의 신뢰도**에 달려 있습니다.

---

## Non-Goals (의도적으로 하지 않는 것)

다음은 현재 Fairpick의 목표가 아닙니다.

- 고도화된 개인 취향 추천 (데이터 축적 이후 고려)
- 모든 이벤트의 포괄적 수집
- AI에 의한 완전 자동 이벤트 생성
- 광고 비용을 기반으로 한 노출 우선 추천

Fairpick은
“많은 이벤트”보다
**“지금 가볼 만한 이벤트”를 정확히 보여주는 것**을 우선합니다.
