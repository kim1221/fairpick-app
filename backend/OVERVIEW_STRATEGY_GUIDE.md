# 카테고리별 Overview 생성 전략 가이드

## 📋 개요

이 문서는 fairpick-app의 Overview 필드에 대한 카테고리별 맞춤 생성 전략을 설명합니다.

**핵심 원칙**:
- Overview는 "이야기 필드"이며 정보 전달용이 아님
- SSOT(Single Source of Truth) 원칙 준수: 날짜/시간/가격/주소는 전용 필드에서만 관리
- 기존 소개글의 의미와 분위기 보존 (완전 새 창작 ❌)
- 카테고리별 맞춤 전략 적용

---

## 🔧 구현 위치

### 파일 경로
```
backend/src/lib/aiExtractor.ts
```

### 핵심 함수
1. **`buildOverviewGuideline(category: string): string`** (Line ~1472)
   - 카테고리별 Overview 작성 가이드라인 생성
   - 5개 카테고리 + 기본 전략 지원

2. **`buildExtractionPromptWithSections(...)`** (Line ~1710)
   - AI 프롬프트 생성 시 `buildOverviewGuideline(category)` 호출
   - 동적으로 카테고리별 가이드 삽입

## 🎨 카테고리별 전략

### 1️⃣ 전시 (EXHIBITION)

**목표**:
- 전시 제목의 의미
- 전시 대상의 역사·문화적 맥락
- 전시 구성 방식

**작성 스타일**:
- 서사형 문장 허용
- "~전시입니다"로만 끝내지 말 것
- 제목의 의미 풀어내기

**Before (❌)**:
```
현대미술 작가 김OO의 개인전입니다. 3층에서 진행됩니다.
```

**After (✅)**:
```
현대미술 작가 김OO이 20년간 천착해 온 '경계'라는 주제를 회화, 설치, 영상 등
다양한 매체로 풀어낸 전시입니다. 관람객은 3층 규모의 공간을 따라 작가의 사유를
따라가며 경계의 의미를 재발견하게 됩니다.
```

**포함 가능 요소**:
- ✅ 작가의 세계관, 작품의 시대적 배경
- ✅ 전시 주제의 역사적 의미
- ✅ 관람 방식 특징 (도슨트, 인터랙티브)
- ✅ 공간 구성 ("3층 규모", "야외 조각정원")

**절대 금지**:
- ❌ 전시 기간 ("3월 15일까지")
- ❌ 입장료 ("성인 15,000원")
- ❌ 운영 시간 ("오전 10시~오후 6시")

---

### 2️⃣ 공연 (PERFORMANCE)

**목표**:
- 장르와 분위기
- 관람 경험의 성격
- 공연의 특징

**작성 스타일**:
- 감정 표현 적극 허용 ("웅장한", "경쾌한")
- 스토리 전체 요약 금지
- 공연 분위기 중심

**Before (❌)**:
```
클래식 음악 공연입니다. 토요일 오후 2시, 7시 2회 공연이며 약 100분 소요됩니다.
```

**After (✅)**:
```
리움챔버오케스트라가 선보이는 실내악 공연으로, 바로크부터 현대까지 아우르는
프로그램으로 구성됩니다. 섬세한 앙상블과 웅장한 오케스트라 사운드를 동시에
경험할 수 있습니다.
```

**포함 가능 요소**:
- ✅ 장르 (클래식, 재즈, 뮤지컬)
- ✅ 출연진 특징 (원로 배우, 젊은 연주자)
- ✅ 음악적/연출적 특성
- ✅ 작품 원작/배경

**절대 금지**:
- ❌ 회차 정보 ("토요일 2회 공연")
- ❌ 러닝타임 ("약 120분")
- ❌ 인터미션 ("15분 휴식")

---

### 3️⃣ 축제 (FESTIVAL)

**목표**:
- 공간 전체 분위기
- 즐기는 방식
- 규모와 지역성

**작성 스타일**:
- 직접 권유 허용 ("즐길 수 있습니다")
- 프로그램 나열 1-2개 핵심만
- 전체 느낌 전달

**Before (❌)**:
```
벚꽃 축제입니다. 3월 20일부터 4월 10일까지 진행되며 무료입니다.
```

**After (✅)**:
```
강변을 따라 펼쳐진 벚꽃길을 걸으며 봄을 만끽하는 지역 축제입니다.
야간 조명으로 밤벚꽃을 감상할 수 있으며, 지역 먹거리 부스와 라이브 공연이
함께하는 활기찬 분위기가 특징입니다.
```

**포함 가능 요소**:
- ✅ 축제 콘셉트 (벚꽃, 음악, 푸드)
- ✅ 대표 프로그램 1-2개
- ✅ 규모감 ("수십 개의 부스")
- ✅ 지역 특색

**절대 금지**:
- ❌ 기간/요일 ("3월 15일부터")
- ❌ 부스 정확 개수 ("부스 47개")
- ❌ 입장료 ("무료 입장")

---

### 4️⃣ 팝업 (POPUP)

**목표**:
- 콘셉트와 체험 포인트
- 브랜드/IP 세계관
- 특별함과 한정성

**작성 스타일**:
- 가볍고 캐주얼한 톤
- 과장 마케팅 절제
- 체험 중심

**Before (❌)**:
```
인기 애니메이션 캐릭터 팝업스토어입니다. 3월까지 운영하며 입장료는 5천원입니다.
```

**After (✅)**:
```
인기 애니메이션 'OOO'의 세계관을 구현한 팝업 스토어로, 캐릭터들이 사는 마을을
재현한 포토존과 작중 등장하는 디저트를 맛볼 수 있는 F&B 공간이 마련되어 있습니다.
팝업 한정 굿즈와 팬아트 전시도 함께 즐길 수 있습니다.
```

**포함 가능 요소**:
- ✅ IP/브랜드 세계관
- ✅ 포토존, 굿즈, F&B
- ✅ 팝업 한정 요소
- ✅ 공간 콘셉트

**절대 금지**:
- ❌ 운영 기간 ("3월까지")
- ❌ 입장료/굿즈 가격
- ❌ 대기 시간

---

### 5️⃣ 행사/강연/포럼 (EVENT)

**목표**:
- 행사 목적과 취지
- 대상과 주제
- 행사 성격

**작성 스타일**:
- 정보 나열체 금지
- 취지와 의도 중심
- 전문성 있지만 딱딱하지 않게

**Before (❌)**:
```
AI 기술에 대한 세미나입니다. 3월 15일 진행되며 참가비는 무료입니다.
```

**After (✅)**:
```
AI 기술이 산업 현장에 미치는 영향을 다루는 전문가 세미나로, 현직 개발자와
연구자들이 실제 적용 사례와 기술적 도전 과제를 공유합니다. AI 분야에 관심 있는
학생과 실무자 모두에게 유익한 자리입니다.
```

**포함 가능 요소**:
- ✅ 주최/주관 단체
- ✅ 핵심 아젠다
- ✅ 대상 (학생, 전문가)
- ✅ 행사 형식 (워크숍, 토론)

**절대 금지**:
- ❌ 일시/장소
- ❌ 참가비
- ❌ 등록 방법
- ❌ 정원/모집 인원

## 🚫 모든 카테고리 공통 금지 사항

### SSOT 위반 (절대 금지)
1. **날짜/시간**: "YYYY년", "X월", "X일", "~부터", "~까지", "약 X분"
2. **가격**: "원", "무료", "유료", "₩"
3. **주소**: 도로명주소, "구/동/로"
4. **링크**: "http", "https", "www"
5. **새로운 사실**: overview_raw나 검색 결과에 없는 내용

### 과도한 마케팅 톤
- "놓치지 마세요", "지금 바로", "역대급", "최고의", "완전 대박"

---

## ✅ 모든 카테고리 공통 허용 사항

### 감성 표현 (과장만 금지)
- "동화 같은", "몰입감 있는", "특별한 경험", "따뜻한"

### 분위기 전달
- "차분한", "활기찬", "신비로운", "경쾌한"

### 세계관 설명
- "~를 따라가며", "~속에서", "~와 함께"

### 객관적 사실
- 주최/출연진/작가 등 핵심 인물
- 대상/체험 방식

## 🧪 검증 기준 (4가지 체크리스트)

모든 Overview는 다음 4가지를 통과해야 합니다:

1. **SSOT 검증**: overview를 읽고 정보 필드(날짜/가격/시간)를 고치지 않아도 되는가?
2. **UI 검증**: 카드에 노출됐을 때 "정보 안내문"처럼 보이지 않는가?
3. **일관성 검증**: 같은 이벤트의 지역/회차 변형에서도 톤과 메시지가 일관적인가?
4. **의미 보존 검증**: 기존 소개글을 아는 사람이 봐도 "의미가 사라지지 않았는가"?

## 🔬 수동 테스트 방법

### 1. Admin UI에서 AI Enrich 테스트

```bash
# 1. Backend 실행
cd backend
npm run dev

# 2. Admin UI 실행 (다른 터미널)
cd backend/admin-web
npm run dev
```

**테스트 절차**:
1. Admin UI 접속: http://localhost:5173
2. 이벤트 선택 (각 카테고리별로 1개씩)
3. "🤖 추천 보완 (네이버+AI)" 버튼 클릭
4. 생성된 Overview 확인

**체크 포인트**:
- 카테고리별 톤이 다르게 적용되었는가?
- SSOT 위반 항목이 없는가? (날짜/시간/가격)
- "~입니다" 1-2문장으로만 끝나지 않았는가?
- 분위기가 느껴지는가?

### 2. 로그로 카테고리 인식 확인

```bash
# Backend 로그 확인
cd backend
tail -f logs/dev.log | grep OVERVIEW
```

**확인 사항**:
- `[OVERVIEW][CATEGORY]` 로그에서 올바른 카테고리 인식 여부
- 카테고리별 다른 프롬프트 적용 여부

### 3. 카테고리별 샘플 테스트

각 카테고리별로 최소 1개씩 테스트:

| 카테고리 | 테스트 이벤트 예시 |
|---------|------------------|
| 전시 | "국립현대미술관 기획전" |
| 공연 | "예술의전당 클래식 공연" |
| 축제 | "여의도 벚꽃축제" |
| 팝업 | "캐릭터 팝업스토어" |
| 행사 | "스타트업 네트워킹 세미나" |

### 4. A/B 비교 테스트

동일 이벤트에 대해:
1. 기존 Overview (이전 버전)
2. 새 Overview (카테고리별 전략 적용)

**비교 항목**:
- 정보성 vs 맥락성
- 단문 요약 vs 서사적 설명
- SSOT 위반 여부
- 분위기 전달 여부

### 5. 검증 스크립트 (선택사항)

```typescript
// backend/src/scripts/test-overview-quality.ts
import { validateOverview } from '../lib/aiExtractor';

const testCases = [
  {
    category: 'exhibition',
    overview: '현대미술 작가 김OO이 20년간...',
    overview_raw: '...',
  },
  // ... more test cases
];

testCases.forEach(tc => {
  const result = validateOverview(tc.overview, tc.overview_raw);
  console.log(`[${tc.category}] Valid: ${result.isValid}`);
  if (!result.isValid) {
    console.log('Violations:', result.violations);
  }
});
```

실행:
```bash
cd backend
npx ts-node -r dotenv/config src/scripts/test-overview-quality.ts
```

## 📊 성공 지표

### 정량적 지표
- SSOT 위반율: 0%
- 평균 문장 수: 3-5문장
- "~입니다" 단문 비율: <30%

### 정성적 지표
- 카테고리별 톤 차별성 (큐레이터 평가)
- 클릭 유도 효과 (A/B 테스트)
- 기존 의미 보존 여부 (원본 대조)

## 🔄 향후 개선 방향

1. **카테고리 세분화**: 전시 → (회화/설치/사진전)으로 세분화
2. **A/B 테스트**: Overview 스타일별 클릭률 측정
3. **사용자 피드백**: "이 설명이 도움이 되었나요?" 수집
4. **자동 검증**: CI/CD 파이프라인에 SSOT 검증 추가
