# Cold Start Problem 해결 방안 요청 프롬프트

## 프로젝트 배경

**Fairpick**: 전시/공연/축제 이벤트 추천 앱 (Toss MiniApp)
- 플랫폼: Apps-in-Toss (토스 슈퍼앱 내 미니앱)
- 기술 스택: React Native (frontend), Node.js + PostgreSQL (backend)
- 데이터 소스: 문화체육관광부 API, KOPIS API, 지역 문화재단 API

---

## 핵심 문제 2가지

### 문제 1: Cold Start Problem
**상황**:
- 앱 출시 초기라서 사용자 행동 데이터가 없음 (조회, 좋아요, 공유 등)
- "지금 핫한 이벤트" 섹션을 채울 방법이 없음
- 초기 사용자들이 들어왔을 때 **"오, 여기 핫한 이벤트들이 모여있네!"** 라는 느낌을 줘야 함
- 빈 섹션이나 무작위 이벤트를 보여주면 신뢰도 하락

**현재 보유한 내부 데이터**:
- `popularity_score`: 이벤트 신선도, 마감 임박도, 소스 우선순위 기반 (화제성과 무관)
- `buzz_score`: 사용자 행동 기반 (초기에는 데이터 없음)

**필요**:
- 초기 단계에서 "진짜 핫한 이벤트"를 찾는 방법
- 사용자 데이터가 쌓이기 전까지 사용할 대안

---

### 문제 2: Admin Hot Discovery (관리자용)
**상황**:
- Admin(나)이 매일 "요즘 핫한 이벤트"를 발견해서 DB에 추가해야 함
- 현재는 일일이 포털 검색, SNS, 커뮤니티를 뒤져야 함 (비효율)

**필요**:
- "요즘 화제가 되고 있는 이벤트" 추천 리스트
- Admin이 리스트를 보고 빠르게 선택하여 수동 입력
- 자동화가 아니라 **추천만** 받으면 됨

---

## 시도했지만 실패한 방법

### 네이버 블로그 검색 API
**시도**:
- 네이버 블로그 검색 API로 이벤트 제목 검색
- `total` (언급 수)를 "화제성" 지표로 사용
- Sampling + 보정 계수로 정확도 개선 시도

**문제**:
- 정확도 **25%** (10개 중 7.5개는 무관한 블로그)
- "뽀로로와 신비한 여행" → 29,005건 (실제 관련은 극소수)
- 보정해도 여전히 **"전혀 핫하지 않은 이벤트가 핫하다고 나옴"**
- 신뢰할 수 없는 데이터

## 제약 조건

### 법적 제약
- ✅ 공공 API 사용: OK (문화체육관광부, KOPIS 등)
- ✅ 크롤링: 사이트 이용약관 위반하지 않는 범위 (robots.txt 준수)
- ❌ 개인정보 수집: NO
- ❌ 저작권 침해: NO (이미지, 텍스트 무단 사용 금지)

### Apps-in-Toss 정책
- ❌ localStorage, sessionStorage 사용 금지
- ❌ 외부 브라우저 열기 제한
- ✅ Storage API 사용 필수 (비동기)
- ✅ 네트워크 요청: OK (백엔드 통해서)

### 기술적 제약
- Backend에서만 외부 API 호출 가능
- Rate limiting 준수 필요
- 실시간성보다는 배치 처리 선호 (1일 1회 업데이트 수준)

## 질문

다음 조건을 **모두** 만족하는 해결 방안을 제시해주세요:

### 1. Cold Start Problem 해결
**목표**: 사용자 데이터가 없는 초기 단계에서 "진짜 핫한 이벤트" 찾기

**요구사항**:
- 법적/정책적으로 문제 없어야 함
- 정확도 70% 이상 (10개 중 7개는 실제 핫함)
- 자동화 가능 (배치 스크립트)
- 비용 효율적 (무료 또는 저렴한 API)

**고려할 데이터 소스 예시** (제한 없음):
- SNS 언급 수 (Twitter, Instagram API?)
- 구글 트렌드 (Google Trends API?)
- 티켓 예매 순위 (인터파크, YES24 크롤링?)
- 포털 실시간 검색어 (네이버, 다음?)
- YouTube 조회수/댓글 수?
- 기타 창의적인 방법

### 2. Admin Hot Discovery
**목표**: Admin이 "요즘 핫한 이벤트"를 빠르게 발견하고 추가

**요구사항**:
- 매일 또는 주간 단위로 "화제 이벤트" 리스트 제공
- Admin이 리스트를 보고 선택하여 수동 입력 (자동 추가 아님)
- 법적/정책적으로 문제 없어야 함

**가능한 워크플로우 예시**:
```
1. 배치 스크립트 실행 (매일 오전)
2. "화제 이벤트" 후보 20개 추출
3. Admin 대시보드에 리스트 표시
4. Admin이 검토 후 "추가" 버튼 클릭
5. 이벤트 정보 수동 입력 (제목, 날짜, 장소 등)
```

## 추가 컨텍스트

### 현재 DB 구조 (canonical_events 테이블)
```sql
- id
- title (이벤트 제목)
- venue (장소명)
- region (지역)
- main_category (전시/공연/축제)
- start_at (시작일)
- end_at (종료일)
- popularity_score (신선도 기반)
- buzz_score (사용자 행동 기반, 초기 0)
- created_at
```

### 목표 사용자
- 20~30대 문화생활 관심층
- 전시/공연/축제 탐색 목적
- "요즘 핫한 것" > "마이너한 것"

### 성공 기준
- 초기 사용자 retention 향상
- "핫한 이벤트" 섹션 클릭률 30% 이상
